## 项目概述

OneResearchTeam 将 OneManCompany（AI 一人公司操作系统）的框架迁移至科研场景，构建一个由 PI（首席科学家）、文献研究员、实验设计师、数据分析师和论文写作专家组成的 AI 科研团队，实现从**文献调研 → 实验设计 → 数据分析 → 论文撰写**的全流程自动化协作。

人类用户扮演 PI 角色，负责审批方向和最终决策；其余角色由 Xiaomi MiMo 大模型驱动，多 Agent 协作完成各阶段任务。

---

## 核心技术

### 1. 多 Agent 协作架构
- **Vessel + Talent 架构**：继承 OneManCompany 的执行容器与技能包分离设计
- **角色定义**：5 个科研角色（PI / LR / ED / DA / WS），各有独立系统提示词和技能
- **任务状态机**：8 阶段科研流程（文献调研 → 方案设计 → 数据采集 → 分析 → 撰写 → 同行评审 → 修订 → 发表）
- **多 Agent 通信**：结构化任务委派、审批链、会议纪要

### 2. Nature Skills 集成
- **scansci-pdf**：论文检索、下载（DOI/arXiv/Sci-Hub）、引文获取
- **Zotero MCP**：文献库管理、引文插入、文献同步
- **OpenAlex API**：学术文献检索（免费，2.4 亿篇论文覆盖）
- **PubMed API**：生物医学文献检索

### 3. 科研工作流引擎
- **阶段门控**：每个阶段完成后需 PI 审批方可进入下一阶段
- **任务队列**：各角色根据阶段自动领取任务
- **知识沉淀**：每次研究积累 SOP 和经验到知识库

## 应用场景

### 当前课题示例：PCN-224@Ag⁺/CMCS 可注射光热水凝胶研究

| 阶段 | 执行角色 | 具体任务 |
|------|---------|---------|
| 文献调研 | LR（文献研究员） | 检索 PCN-224 / 光热抗菌 / 猪 MRSA 模型相关文献，整理关键发现和研究空白 |
| 实验方案设计 | ED + PI | 设计猪 MRSA 感染创面模型实验方案，分组、评价指标、SOP |
| 数据采集 | ED | 执行实验方案，记录原始数据 |
| 数据分析 | DA + PI | 统计分析、显著性判断、图表制作 |
| 论文撰写 | WS + PI | 按目标期刊格式撰写SCI论文 |

### 可扩展领域
- 兽医药理（当前方向）
- 纳米医学 / 药物递送
- 动物感染模型
- 任何依赖文献-实验-数据-写作流程的学科

## 主要功能

- **文献管理**：检索、下载、整理、引用自动化
- **实验设计**：方案生成、伦理审批、SOP 导出
- **数据分析**：统计方法选择、图表制作、质量控制
- **论文写作**：多期刊格式支持、Cover Letter、审稿意见回复
- **团队协作**：角色分工、审批流、进度追踪
- **知识积累**：研究 SOP 沉淀、模板复用

## 技术栈

- **AI 驱动**：Xiaomi MiMo 大模型（多 Agent 推理）
- **底层框架**：OneManCompany（开源，Apache-2.0）
- **语言**：Python 3.12+
- **文献工具**：scansci-pdf MCP / Zotero MCP
- **数据处理**：Python（pandas / scipy / matplotlib）

## 预期产出

1. **科研流程效率提升**：减少重复性文献整理和数据统计工作时间
2. **标准化 SOP 知识库**：形成可复用的实验设计和论文写作模板
3. **多课题并行管理**：可同时支持 3-5 个研究课题并行推进
4. **SCI 论文产出**：以兽医药理 MOF 光热抗菌方向为验证场景
